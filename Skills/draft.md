# Skill: Draft

## Purpose
Create initial drafts of documents, emails, reports, and other written content based on requirements and context.

## Inputs
- Document type: Email/Report/Proposal/Article/etc.
- Purpose: What the document should achieve
- Audience: Who will read it
- Key points: Main messages to convey
- Tone: Formal/Casual/Technical/etc.
- Length: Target word count or page count

## Outputs
- Draft document: Initial version ready for review
- Metadata: Word count, reading time, tone assessment
- Suggestions: Optional improvements or alternatives
- Review checklist: Items to verify before approval

## Process

1. **Understand Requirements**
   - Clarify document purpose and goals
   - Identify target audience
   - Gather necessary information and context
   - Review similar past documents

2. **Structure Content**
   - Create outline with main sections
   - Organize key points logically
   - Plan introduction and conclusion
   - Identify supporting details needed

3. **Write Draft**
   - Follow appropriate format and style
   - Match tone to audience and purpose
   - Include all required information
   - Use clear, concise language

4. **Enhance Quality**
   - Add relevant examples or data
   - Ensure logical flow
   - Check for completeness
   - Verify accuracy of facts

5. **Prepare for Review**
   - Format properly
   - Add metadata and tags
   - Move to /Pending_Approval
   - Include review checklist

## Success Criteria
- Draft addresses all requirements
- Appropriate tone and style
- Clear and well-organized
- Factually accurate
- Ready for human review with minimal edits

## Error Handling
- **Insufficient information**: Request additional context, create draft with placeholders
- **Unclear requirements**: Ask clarifying questions before drafting
- **Conflicting guidelines**: Flag conflicts, propose resolution
- **Complex topic**: Break into sections, draft incrementally

## Examples

### Example 1: Email Draft
**Input:**
- Type: Professional email
- Purpose: Request meeting with client
- Audience: Senior executive
- Key points: Discuss Q2 strategy, review metrics
- Tone: Professional but friendly

**Output:**
```markdown
Subject: Q2 Strategy Discussion - Meeting Request

Hi [Client Name],

I hope this message finds you well. As we approach Q2, I'd like to schedule time to discuss our strategic priorities and review the metrics from Q1.

I'm particularly interested in exploring:
- Q2 goals and key initiatives
- Performance analysis from Q1
- Resource allocation for upcoming projects

Would you have 30-45 minutes available next week? I'm flexible on timing and happy to work around your schedule.

Looking forward to connecting.

Best regards,
[Your Name]
```

**Review Checklist:**
- [ ] Recipient name filled in
- [ ] Specific dates proposed
- [ ] Contact information included
- [ ] Tone appropriate for relationship

### Example 2: Project Proposal
**Input:**
- Type: Project proposal
- Purpose: Get approval for new initiative
- Audience: Management team
- Key points: Problem, solution, budget, timeline
- Tone: Professional, persuasive

**Output:**
```markdown
# Project Proposal: Customer Portal Enhancement

## Executive Summary
Propose upgrading customer portal to improve user experience and reduce support tickets by 30%. Estimated investment: $25k over 3 months.

## Problem Statement
Current portal has 45% bounce rate and generates 200+ support tickets monthly. User feedback indicates navigation confusion and missing self-service features.

## Proposed Solution
Redesign portal with:
- Intuitive navigation structure
- Self-service knowledge base
- Account management tools
- Mobile-responsive design

## Expected Outcomes
- 30% reduction in support tickets
- 50% improvement in user satisfaction scores
- 20% increase in portal engagement
- ROI within 6 months

## Budget
- Design: $8,000
- Development: $12,000
- Testing: $3,000
- Contingency: $2,000
**Total: $25,000**

## Timeline
- Month 1: Design and planning
- Month 2: Development
- Month 3: Testing and launch

## Next Steps
1. Approve budget and timeline
2. Assign project team
3. Begin design phase

## Risk Assessment
- Timeline risk: Medium (mitigated by phased approach)
- Budget risk: Low (includes 8% contingency)
- Technical risk: Low (proven technologies)
```

**Review Checklist:**
- [ ] All budget items justified
- [ ] Timeline realistic
- [ ] Success metrics defined
- [ ] Risks identified and mitigated
